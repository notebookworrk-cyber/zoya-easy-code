# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please:

1. **Do not** open a public issue
2. Email the maintainers or open a [security advisory](https://github.com/notebookworrk-cyber/zoya-easy-code/security/advisories/new)
3. Provide a detailed description and steps to reproduce

We will:
- Acknowledge receipt within 48 hours
- Provide a timeline for the fix within 5 business days
- Notify you when the vulnerability is fixed

## Scope

- The Zoya interpreter and standard library
- CLI tools and developer utilities
- Build and deployment pipelines

## Out of Scope

- Dependencies managed by pip (report to their respective maintainers)
- The Python runtime itself

Thank you for helping keep Zoya secure.
